# HELLGRID — ブラウザ レイキャスティングFPS

依存ライブラリなし・素のCanvas+JavaScriptで動くレイキャスティングFPSのデモです。
グラフィック・サウンドはすべてコードで手続き生成したオリジナルで、外部アセットは
一切使っていません(特定の既存ゲームとは無関係のファンメイド習作です)。

## 遊び方

`index.html` をブラウザで開くだけで動きます(サーバー不要、ダブルクリックでOK)。

| 操作 | キー |
|---|---|
| 移動 | WASD |
| 旋回・上下視点 | マウス(クリックでロック)/ ←→ |
| 射撃 | クリック / Space |
| ドア・出口スイッチ | E |
| 武器切替 | 1(ピストル)/ 2(ショットガン)/ 3(ナイフ) |
| ミニマップ | M |
| バックミラー(後方視界) | B |
| サウンドON/OFF | P |
| タイトルへ戻る(死亡時) | Q |
| **AIにプレイさせる** | **I** |

### AI デモ

**▶ ブラウザですぐ見る: https://taikiishii.github.io/hellgrid/** (タイトル画面で I キー)

強化学習(PPO)で学習したAIが自動でプレイします。プレイ中に I を押せば、その場で
AIに交代したり操作を取り戻したりできます。

| 操作 | キー |
|---|---|
| AIにプレイさせる / 交代 | I |
| **AIの「見え方」を可視化** | **V** |
| 再生速度(0.1〜4倍) | `[` / `]` |
| 一時停止 / コマ送り | `\` / `.` |
| リプレイを保存 / 読み込む | O / L |

**V キーの可視化**が一番おもしろいところです。エージェントが実際に受け取っている
観測(1231次元)をそのまま画面に描きます。

- **上のバー** … レイ24本。棒の高さ=壁までの近さ、赤=敵、緑/青=アイテム
- **右のグリッド** … 自己中心11×11。**緑=出口に近づく向き / 暗赤=遠ざかる向き**、
  赤い四角=敵、黄=ドア、白い三角=自分
- **下のストリップ** … いまAIが「押している」入力

緑の方向へ一直線に走り、赤い敵を素通りしていく様子がそのまま見えます。

推論はブラウザ内で完結しています。外部ライブラリもサーバーも不要で、`index.html` を
ダブルクリックするだけで動きます。方策(90万パラメータのMLP)は `js/policy.js` に
重みを埋め込んであり、行列積は `js/ai.js` が素のJavaScriptで回しています。

**リプレイ**は「シード + 行動列」だけで保存します(1ステージ2.4KB)。シミュレーションが
決定的なので、位置もHPも時刻も寸分違わず再現されます。

現在のAIは**全5ステージを平均9.1秒でクリア**します(録画: `media/ai-play.webm`)。
ただし敵はほとんど倒しません(倒すより走り抜けるほうが報酬が高いと学習したため)。
また、マップの探索もしていません(出口までの最短経路を観測として与えているため)。
この線引きについては [docs/machine-learning.md](docs/machine-learning.md) を参照。

緑の「EXIT」スイッチを見つけてEキーで起動するとステージクリア。
現在は5ステージ(E1M1: 侵入、E1M2: 闘技場、E1M3: 処理施設、E1M4: 監視塔、
E1M5: 水路)。

### モード(カラーテーマ)

タイトル画面で **←→ または T キー**でモードを切り替えられます(プレイ中の
切り替えも可)。ゲームの中身は同じで、見た目だけが変わります。

- **地獄 (ホラー)**: 元々のダークな世界観。亡兵・焔鬼・牙獣など。
- **ほのぼの (パステル)**: パステルカラーの空と芝生。敵は子ねこ・うさぎ・
  まんまるブタ・くまなどのほのぼの動物に置き換わり、武器は水でっぽう、
  火球はシャボン玉、攻撃は「ぽふっ」とした煙になります(流血表現なし)。

- **ロックドア**: 赤/青の帯が入ったドアは対応するキーカードが必要。
  取得状況はHUD右下の KEYS に表示される。キーはステージ毎にリセット。
- **隠し扉**: 一見ただの壁でもEキーで開く場所がある。リザルトの
  「シークレット」にカウントされる。怪しい壁は調べてみよう。
- **アンブッシュ**: アイテムを取ると罠が発動して敵がテレポートして
  くることがある。「罠だ!」と出たら戦闘準備。
- **ナイフ(近接)**: 3キーでいつでも装備でき、弾を消費しない。距離ゼロ
  (敵に密着)でしか当たらないが攻撃力は高め(牙獣を数発で倒せる)。
  銃の残弾が尽きると自動でナイフに持ち替わる。ほのぼのモードでは
  「肉球パンチ」になる。
- **段差・高台(全ステージ)**: 階段で高台に登ったり、テラスから敵を
  見下ろして撃ったりできる。マウスを上下に動かすと視点が上下に振れる
  (高い敵を狙うときに使う)。高低差0.5タイルまでは歩いて登れ、それ
  より高い段差は壁になる。敵も同じルールで動き、高所からは飛び降りない。
- **水路・池 / 動く歩道(E1M5〜)**: 水路・池は通れない窪み(弾は上を
  通過する)。動く歩道に乗るとその向きへ流される。どちらもプレイヤー・
  敵の両方に効く。
- **爆発樽**: 撃つと誘爆し、爆風で周囲の敵を巻き込む(近すぎると自分も
  ダメージ)。樽が連鎖して大爆発することも。敵の火球が当たっても爆発する。
- **飛行する敵(漂霊)**: 空中を漂い、段差や水を無視して飛ぶ。遠距離から
  弾を撃ち下ろしてくる。マウスの上下視点で狙う。
- **リフト**: 時間で上下する床。低い所と高い所を結ぶ。上がってくるのを
  待って乗る。
- **バックミラー**: B キーで画面上端に後方視界の小窓を表示。振り向かずに
  背後から迫る敵を把握できる(壁に隠れた敵は映らない)。もう一度 B で消す。
- **アーマー**: 拾うと被ダメージの約1/3を肩代わりしてくれる(残量が尽きる
  まで)。HUDの ARMOR に残量が表示される。`p`=破片(+5)、`V`=ボディ
  アーマー(+100)、上限200。HPと同様ステージをまたいで持ち越す。

|---|---|
| 亡兵 / 子ねこ | 単発射撃。弱い |

## ファイル構成

シミュレーション(ゲームのルール)と、描画・音・入力(見た目)を分けてあります。
シミュレーション側はDOMにもCanvasにも実時間にも依存せず、Nodeでヘッドレスに
そのまま走ります(機械学習で自動プレイさせるため)。

```
index.html          … エントリ(canvasとスクリプト読み込みのみ)
js/levels.js        … ステージデータ(マップ・敵・アイテム配置)

js/sim/             ── シミュレーション: DOM非依存・決定的 ──
  rng.js            … シード付き擬似乱数(Math.randomは使わない)
  constants.js      … 定数と敵・武器・アイテムのテーブル
  world.js          … World クラス(移動・戦闘・敵AI・ドア・状態遷移)

js/themes.js        … カラーテーマ(見た目だけ)
js/sound.js         … WebAudio手続き生成サウンド
js/render.js        … テクスチャ・スプライト・レイキャスト描画・HUD
js/main.js          … ブラウザ用シェル(入力・タイトル・固定タイムステップのループ)

js/ai.js            … AIデモ(方策のMLPを素のJSで推論し、Worldに入力を流す)
js/policy.js        … 学習済みの重み(自動生成。tools/export-policy.py)

env/                ── 機械学習 (強化学習の環境と学習) ──
  sim-loader.cjs    … Nodeから js/sim/* を読み込む(vm・ビルド不要)
  obs.js            … 観測(レイ・自己中心グリッド・出口までのBFS距離場)
  env.js            … HellgridEnv(行動・報酬・エピソード管理)
  server.js         … 1プロセスでN環境をホストするバッチサーバ(stdio)
  protocol.py       … server.js と話す低レベル層(numpyのみ)
  hellgrid_env.py   … Stable-Baselines3 の VecEnv
  train.py          … PPO + カリキュラム
  eval.py           … 学習した方策の評価
```

`World` は1インスタンス=1ゲーム世界で、同一プロセスに何個でも並べられます。
音・メッセージ・ポインタロックは `emit()` でイベントとして外に投げるだけで、
`World` 自身は副作用を持ちません。

### テスト

```
node tools/check-maps.js    … マップデータ検証(行の長さ・キーの整合性・高さレイヤー)
node tools/sim-test.cjs     … ヘッドレスでシムを検証(決定性・スループット)
node tools/env-test.cjs     … 学習環境を検証(観測の値域・BFS距離場・報酬整形)
python tools/smoke-test.py  … ブラウザで起動して描画とコンソールエラーを確認
python tools/parity-test.py … Nodeのシムとブラウザのシムが完全一致するか検証
python tools/bench-env.py   … 学習環境のスループット計測(SB3不要)

python tools/policy-parity.py … PyTorchの推論とJSの推論が一致するか検証
python tools/ai-demo-test.py  … ブラウザでAIにプレイさせ、実際にクリアできるか確認
```

**Nodeで学習した方策がブラウザでそのまま動くこと**は、2つのテストで保証しています。

- `parity-test.py` … シミュレーションの一致(同じシード・同じ入力 → 同じ状態列)
- `policy-parity.py` … 推論の一致(同じ観測 → 同じ行動。ロジットの差 1e-6 未満)

## 機械学習で自動プレイさせる

強化学習(PPO)で、全5ステージをクリアする方策を学習させます。

**→ 設計と実装の詳しい解説: [docs/machine-learning.md](docs/machine-learning.md)**
(強化学習が初めての人向け。観測・報酬設計・カリキュラム・実際に踏んだバグまで)

### 環境の設計

このゲームはレイキャスターなので、**画面を描いてCNNに見せる必要がありません**。
レイが何に当たったかを直接ベクトルにすれば「見えているもの」とほぼ等価な観測になり、
描画コストがゼロになるぶん学習が2〜3桁速くなります。

- **観測** `Box(1231,)`
  - レイ 24本 × 15ch: 壁までの距離、ドア/施錠ドア/出口スイッチか、そのレイ上の
    敵の距離と種別、アイテムの距離と種別、樽の距離
  - 自己中心グリッド 11×11 × 7ch: 壁・ドア・水・高さ・敵・アイテム、そして
    **出口までのBFS距離の勾配**(これが探索の背骨になる)
  - スカラー 24: HP・アーマー・弾・武器・キーカード・出口までの距離 など
- **行動** `MultiDiscrete([3,3,5,3,2,2,4])` — 前後 / 左右 / 旋回 / 上下視点 /
  射撃 / E(使う) / 武器切替。フレームスキップ4で15Hzで判断
- **報酬** 与ダメージ・キル・アイテム・キーカード・シークレット・ステージクリアを正、
  被ダメージ・死亡・時間・弾の消費を負。加えて**出口までのBFS距離によるポテンシャル
  整形**(1タイル近づくごとに +0.1)。これがないと疎報酬でナビゲーションが学習できません
- 出口が施錠ドアの向こうにあるステージでは、目標が自動的に**キーカードに切り替わり**、
  取った瞬間に出口へ切り替わります

### 学習

```
python -m venv .venv && .venv/Scripts/pip install torch "stable-baselines3[extra]"

python env/train.py --stage nav      --steps 3000000                                # 敵なし・E1M1で「出口に着く」
python env/train.py --stage nav-all  --steps 5000000  --init runs/nav/final.zip     # 敵なし・全5ステージ
python env/train.py --stage combat   --steps 5000000  --init runs/nav-all/final.zip # E1M2で戦闘
python env/train.py --stage single   --steps 20000000 --init runs/combat/final.zip  # 全部入り・1ステージ単位
python env/train.py --stage campaign --steps 20000000 --init runs/single/final.zip  # E1M1から通しでクリア

python env/eval.py --model runs/single/final.zip --stage single --episodes 50
tensorboard --logdir runs/
```

いきなり全部入りで学習させても、序盤の報酬が疎すぎて学習が立ち上がりません。
上のように段階を追って、前段の重みを `--init` で引き継ぎます。

### 性能

シミュレーションは1コアで **46k step/秒**(描画なし)。観測の生成を含めると
**3.5k 行動/秒**。Node は単スレッドなので、12ワーカー × 32環境に分けて
**8.8k 行動/秒(35k sim step/秒)** 出ます。PPO の学習は 7k FPS 前後で回ります。

## ステージの追加方法

`js/levels.js` の `LEVELS` 配列にオブジェクトを足すだけです。マップは文字列の
配列で、1文字=1タイル。凡例は `levels.js` 冒頭のコメント参照:

- 壁: `#` レンガ / `&` テック / `=` 石柱 / `D` ドア / `R` `B` ロックドア /
  `*` 隠し扉 / `X` 出口スイッチ
- 床: `.` / `P` 開始位置 / `T` アンブッシュトリガー /
  `~` 水路・池(通行不可) / `>` `<` `^` `v` 動く歩道(その向きへ流す) /
  `L` リフト(時間で上下) / `o` 爆発樽
- 敵(続き): `F` / `f` 漂霊(飛行。`heights` で高い位置に置くと映える)
- 敵: `Z` 亡兵 / `G` 散弾兵 / `I` 焔鬼 / `M` 牙獣 / `K` 獄騎士
  (小文字 `z g i m k` は休眠状態で配置され、トリガーで出現)
- アイテム: `h H` 回復 / `a A` 弾 / `s` シェル / `S` ショットガン /
  `r b` キーカード / `p` アーマー破片 / `V` ボディアーマー

### 立体ステージ(段差・高台)

オブジェクトに任意フィールド `heights` を足すと床に高さがつきます。
マップと同じ縦横サイズの文字列配列で、各文字 `0`〜`9` がそのタイルの
床の高さ(`0.25` タイル刻み、`4` = 1タイルぶんの高台)。階段は
`0→1→2→3→4` のように1マスずつ刻みます(隣との差が `2`=0.5 を超えると
壁扱いになり登れません)。あわせて `ceilHeight`(天井の高さ・整数、高台を
作るなら `2` 推奨)と `stepTex`(段差側面のテクスチャ文字、既定 `=`)も
指定できます。詳しくは `levels.js` 冒頭コメントと E1M4 のデータを参照。

追加したら `node tools/check-maps.js` で行の長さ・外周の閉じ忘れ・
キーの整合性(Rドアがあるのにrキーがない等)、高さレイヤーのサイズと
文字を検証できます。

## 拡張ポイント(さらに育てるためのメモ)

データ駆動なので、以下はテーブルにエントリを足すだけで増やせます:

- **敵の種類** … `sim/constants.js` の `ENEMY_TYPES` + `ENEMY_CHARS` +
  `render.js` にスプライト描画関数
- **武器** … `sim/constants.js` の `WEAPONS` テーブル(例: チェインガン=低ダメージ
  高連射。`melee: true` を付けると近接武器になり、`range`/`halfWidth` で間合いを指定)
- **アイテム** … `sim/constants.js` の `ITEM_TYPES`(例: 無敵スフィア)
- **壁テクスチャ** … `render.js` の `buildTextures()` に文字を追加
- **カラーテーマ** … `themes.js` の `THEMES` に色パレット1つ + `SPRITE_BUILDERS` に
  敵スプライト描画関数のセット1つを足すと、新しいモードが選べるようになる
  (壁・床・UI・武器・敵・エフェクトの色と見た目をまとめて差し替え)

仕組みの追加が必要な候補:

- 樽の爆発(爆風で敵も巻き込む)
- 飛行する敵(浮遊するタイプ。`heights` を活かして高い位置に配置)
- リフト(時間で上下する床)、セーブ機能、難易度設定
- BGMのステージ別バリエーション
